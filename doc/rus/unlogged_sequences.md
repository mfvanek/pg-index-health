# Проверка наличия нелогируемых последовательностей

Нелогируемые последовательности (UNLOGGED) не поддерживаются механизмом Write-Ahead Log (WAL),
поэтому их состояние не реплицируется на резервные серверы и сбрасывается при аварийном
завершении работы сервера. Потеря текущего значения последовательности может привести к
ошибкам дублирования ключей, если она используется как значение по умолчанию для столбца.

Смотри также https://www.postgresql.org/docs/current/sql-createsequence.html

## SQL запрос

- [unlogged_sequences.sql](https://github.com/mfvanek/pg-index-health-sql/blob/master/sql/unlogged_sequences.sql)

## Тип проверки

- **static** (может выполняться на пустой БД в компонентных/интеграционных тестах)

## Поддержка секционированных таблиц

Не применима к секционированным таблицам (последовательности — самостоятельные объекты).

## Скрипт для воспроизведения

```sql
create schema if not exists demo;

create unlogged sequence demo.unlogged_seq_1;
```

## Как исправить

Преобразуйте нелогируемую последовательность в обычную (логируемую):

```sql
alter sequence demo.unlogged_seq_1 set logged;
```
